# Public-Sector UI Review Checklist

- [ ] KRDS 또는 기관 공통 UI 기준을 따라야 하는지 확인했는가
- [ ] 기관 공통 템플릿, Tiles/layout, 공통 include, 공통 JS/CSS 의존 범위를 확인했는가
- [ ] 화면 구조가 공공서비스형 정보구조와 크게 어긋나지 않는가
- [ ] 제목, breadcrumb, section heading, 버튼 라벨이 일관적인가
- [ ] 필수 입력, 도움말, 에러 메시지, 검증 메시지 기준이 정리되었는가
- [ ] 키보드 접근성, focus 이동, 명도 대비, semantic markup 기본 점검을 했는가
- [ ] 첨부파일, 표, 검색조건, 결과 목록, 상세/수정 폼 패턴을 검토했는가
- [ ] 공통컴포넌트나 기관 공통 화면 패턴을 건드리는 수정이라면 before/after parity를 별도로 적었는가
- [ ] 개인정보 또는 민감정보가 화면/로그/다운로드에 과노출되지 않는가
- [ ] before/after 비교가 필요한 UI라면 parity 기준과 스크린샷 또는 수동 검증 항목이 있는가
